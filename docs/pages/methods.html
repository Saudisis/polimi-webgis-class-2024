Methodology

The methodology integrates multiple data sources and analytical techniques to assess landslide susceptibility comprehensively.

Workflow

Our approach includes a comprehensive four-step process: data preprocessing for susceptibility mapping, susceptibility map generation, data preprocessing for exposure assessment and exposure assessment. This systematic methodology ensures precise and reliable results in our geographic information system (GIS) project.

The Normalized Difference Vegetation Index (NDVI) is a numerical measure used to evaluate vegetation health and vitality. NDVI is derived from the reflectance values of near-infrared (NIR) and red light bands. This index calculates the difference between the reflectance of vegetation (which strongly reflects NIR) and non-vegetation surfaces (which reflect both NIR and red light). NDVI values range from -1 to +1, with higher values representing denser and healthier vegetation cover.

DUSAF forms the basis for analyzing urban patterns, quantifying the urban heat island effect, and assessing urban sprawl.

Buffer layers around rivers are generated by extending the original river lines outward by a defined distance, creating a new polygon layer that encloses the rivers. This allows for spatial analysis to examine the relationships between rivers and other features, such as landslides. By analyzing these buffer zones, we can identify areas within specific distances from rivers, assess potential impacts, and gain insights into the interactions between water bodies and the landscape.

Data Processing

Terrain analysis and characterization were performed using the slope, aspect, and curvature tool, starting from the DTM layer. This was done with QGIS, specifically using the SAGA tool. Additionally, the population layer was added for subsequent analysis.

The slope tool evaluates the incline of the terrain across a raster surface, determining how steep it is at different points. It gauges the elevation shift between adjacent cells and presents this as either a percentage or angle measurement.

Slope

  • Details

The aspect tool identifies the cardinal direction that a slope is oriented towards at every point on a raster surface. It reveals the terrain's orientation, indicating whether it faces north, south, east, or west. Analyzing aspect is valuable for assessing how slopes are exposed to sunlight and wind patterns.

Aspect

  • Details

Profile curvature reflects the curvature of the slope on its vertical profile. It distinguishes between concave and convex terrain features, which are critical factors in landslide formation. Concave terrain tends to trap water, increasing erosion and instability, while convex terrain sheds water more efficiently, reducing these risks.

This information is vital for assessing landslide risk, as concave terrain can accumulate water, leading to erosion and instability, while convex terrain tends to shed water, reducing such risks.

Processing No Landslides Zones

Utilizing the Slope layer generated in the previous step, we designate areas with low landslide probability as No Landslide Zones (NLZ). Following the next steps:

  1. NLZ areas are defined as slopes with angles smaller than 20° or greater than 70°, employing the raster calculator.
  2. To refine NLZ delineation and eliminate small pixel patches, the sieve tool is applied with a filter set at 150.
  3. Although the NLZ layer is generated with some approximation, overlaps with the Landslide Inventory (LS) layer occur. To address this, the Difference tool is used to remove LS overlapping portions from the NLZ layer.

Train and testing points labels assignment

A Hazard feature was created to differentiate No Landslide Zones from Landslide Zones: No Landslide Zones: 1 , Landslide Zones: 2

The training-testing ratio used to train the Machine Learning Model is 70/30. Therefore, 70% of LZ and NLZ polygons were randomly selected and assigned the "Training" label, while the remaining polygons were assigned the "Testing" label. After this process, the hazard and training/testing labels were consolidated into one attribute table containing all polygons of LZ and NLZ.

Virtual multilayer generation

Virtual Multilayer Raster

All environmental factors were meticulously combined into a single raster layer using the Build Virtual Raster tool in QGIS. This process involved integrating various datasets such as terrain elevation, vegetation indices, land use, roads, rivers, and fault lines to create a unified representation. This comprehensive raster layer was then prepared for input into the Dzetsaka classification tool, enabling a more sophisticated analysis and accurate classification of landslide susceptibility in the Valdisotto region.

Comparison with other classifiers

Using this dataset, we tested two additional classification methods to evaluate their overall accuracy and determine the best susceptibility map

K-Nearest Neighbor (KNN)

1st alternative

For each point in the raster, the KNN algorithm considers the susceptibility levels of the nearest points in the training dataset. It assigns the most common susceptibility level among these neighbors.

Data preprocessing for exposure assessment

Processing population dataset

Population count data sourced from the WorldPop research programme served as the foundational dataset for this analysis. Specifically, the retrieval criteria targeted the total population count for Italy in the year 2020. Upon acquisition, the dataset was found to possess a spatial resolution of 100 meters, with each pixel on the matrix representing a quantifiable number of individuals. Subsequently, the map underwent a series of geospatial transformations, including reprojection and clipping, tailored to the study area's specifications.
